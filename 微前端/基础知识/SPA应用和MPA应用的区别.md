# SPA 应用与 MPA 应用的区别

SPA（Single Page Application，单页应用）和 MPA（Multi Page Application，多页应用）是目前前端开发的两种主流架构模式，它们的区别非常明显，适用于不同的项目场景。下面从多个维度详细对比：

| 对比维度           | SPA（单页应用）                              | MPA（多页应用）                              |
|---------------------|---------------------------------------------|---------------------------------------------|
| **页面切换方式**    | 不刷新整个页面，通过 JavaScript 动态替换内容（路由通常是客户端路由） | 每次切换页面都向服务器请求完整的 HTML，浏览器整页刷新 |
| **代表技术栈**      | React + React Router、Vue + Vue Router、Angular、SvelteKit（SPA 模式）等 | 传统服务端渲染：PHP（Laravel/ThinkPHP）、Java（Spring MVC）、Python（Django/Flask）、Node.js（Express + EJS）等 |
| **首屏加载速度**    | 通常较慢（需要先下载一个较大的 JS Bundle） | 通常较快（每个页面只加载当前所需资源）      |
| **后续页面切换速度**| 极快（只请求数据，DOM 局部更新）            | 较慢（每次都要完整请求 + 渲染）             |
| **SEO（搜索引擎优化）** | 天然不友好（早期完全靠 JS 渲染，爬虫抓不到内容）<br>现代解决方案：SSR（Next.js、Nuxt3）、Prerendering、SSG | 天生友好（服务器直接返回完整 HTML）         |
| **开发复杂度**      | 前端路由、状态管理、代码分割难度较高         | 每个页面相对独立，开发逻辑更简单直观        |
| **服务器压力**      | 小（主要返回 JSON 数据，静态资源可 CDN）     | 大（每次页面切换都要服务器渲染完整 HTML）   |
| **用户体验**        | 接近原生 App，流畅、动画丰富                 | 传统网页体验，有白屏和刷新感                |
| **典型应用场景**    | 后台管理系统、复杂交互的 Web 应用（如 Gmail、Trello、Figma、Notion） | 官网、博客、电商商品列表页、新闻门户、企业站、需要强 SEO 的内容站 |
| **打包后资源**      | 通常只有一个 index.html + 一堆 JS/CSS chunk | 每个路由对应一个完整的 HTML 文件            |

### 更直观的例子对比

| 项目               | SPA 例子                               | MPA 例子                              |
|-------------------|----------------------------------------|---------------------------------------|
| 典型网站           | https://twitter.com（现 X.com）         | https://www.baidu.com                 |
| 现代框架实现方式   | Create React App、Vite + React Router   | Next.js（默认是文件系统路由，App Router 也可以做 MPA）<br>Nuxt3、Remix |
| 路由跳转时地址栏变化 | 是（History API）                      | 是（传统 a 标签或表单提交）           |
| 后退/前进按钮体验  | 完美支持                               | 完美支持                              |

### 当前主流趋势

1. **纯 SPA 正在减少**：因为 SEO 和首屏性能问题，纯客户端渲染的 SPA 越来越少。
2. **SSR/SSG 成为主流**：
   - Next.js（React）→ 默认是服务端渲染 + 静态生成，可以做到“看起来像 SPA，实际上是 MPA/SSG”
   - Nuxt3（Vue）
   - SvelteKit、Remix、Astro 等都支持混合模式
3. **实际项目中往往是混合的**：
   - 营销页、博客、详情页 → 用 SSG/MPA（静态生成或服务端渲染）
   - 管理后台、复杂交互页面 → 用 SPA（客户端路由）

### 总结一句话区别

- **SPA**：一次加载，之后全靠前端 JS 控制路由和渲染，像桌面应用。
- **MPA**：每次换页面都重新向服务器请求一个完整的新页面，像传统网站。

选择建议：
- 需要极致交互体验 + 不care SEO → 选纯 SPA
- 需要 SEO + 首屏快 + 内容型网站 → 选 MPA 或现代 SSR/SSG 方案（推荐 Next.js/Nuxt）
- 大型项目 → 通常是“外层 MPA（营销页）+ 内层 SPA（核心应用）”的混合架构

这就是目前前端开发中最核心的架构选择区别了。
