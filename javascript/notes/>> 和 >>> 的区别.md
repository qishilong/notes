# >> 和 >>> 的区别

**JavaScript 中 `>>` 和 `>>>` 的区别**

JavaScript 中的 `>>` 和 `>>>` 都是**位右移运算符**，它们会把操作数的二进制表示向右移动指定的位数。  
两者在处理**正数**时行为完全相同，但处理**负数**时有关键区别，因为 JavaScript 的位运算把数字视为 **32 位有符号整数**（使用二补码表示）。

| 运算符 | 名称                               | 对正数的表现       | 对负数的表现                          | 左侧填充方式         |
| ------ | ---------------------------------- | ------------------ | ------------------------------------- | -------------------- |
| `>>`   | 带符号右移（Signed right shift）   | 正常右移，左侧补 0 | 保留符号位，左侧用 1 填充（保持负数） | 根据原数的符号位填充 |
| `>>>`  | 无符号右移（Unsigned right shift） | 正常右移，左侧补 0 | 总是左侧补 0，结果变为正数（很大）    | 总是补 0             |

## 关键区别总结
- `>>` **保留符号位**：负数右移后仍然是负数。
- `>>>` **忽略符号位**：把整个 32 位视为无符号数，左侧永远补 0，负数右移后会变成一个很大的正数。

## 代码示例（推荐在浏览器控制台或 Node.js 中运行验证）

```js
// 正数示例（两者相同）
console.log(16 >> 2);   // 4   (16 的二进制 000...010000 → 右移2位 → 000...000100 → 4)
console.log(16 >>> 2);  // 4

console.log(8 >> 1);    // 4
console.log(8 >>> 1);   // 4

// 负数示例（区别明显）
console.log(-16 >> 2);  // -4   （保留符号位，左侧补1）
console.log(-16 >>> 2); // 1073741820   （左侧补0，变成大正数）

console.log(-8 >> 1);   // -4
console.log(-8 >>> 1);  // 2147483644   （2^31 - 4）

console.log(-1 >> 1);   // -1   （全1右移后左侧补1，仍是全1）
console.log(-1 >>> 1);  // 2147483647   （全1右移后左侧补0，得到 0111...1）
```

## 实际应用场景
- `>>`：常用于需要保留符号的整数除法（相当于 `/ 2^n` 并向下取整），或有符号整数的位操作。
- `>>>`：常用于把有符号数强制转为无符号数处理（如颜色值、Hash 计算、某些加密算法），或需要把负数变成大正数的场景。

简单记住：**有负数时用 `>>` 保持负号，用 `>>>` 会把负数变成大正数**。
