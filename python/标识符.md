# 标识符

在 Python 中，**标识符（identifier）** 和 **变量（variable）** 是两个经常被混淆的概念，但它们完全不是一回事。下面用最直接、最准确的方式给你讲清楚它们的区别：

| 项目             | 标识符 (Identifier)                                | 变量 (Variable)                                      |
|------------------|----------------------------------------------------|-----------------------------------------------------|
| 本质             | 就是一个「名字字符串」                              | 是「名字」和「对象」在运行时的绑定关系               |
| 属于哪个层面   | 语法层面、静态的、写代码时就确定了                 | 运行时层面、动态的、程序执行到赋值时才产生           |
| 是否必须绑定对象 | 不需要（可以只是声明）                              | 必须绑定一个对象（否则使用会报 NameError）           |
| 作用范围         | 可以给变量、函数、类、模块、包等任何实体命名        | 通常只指「能被赋值、重新赋值」的名字（可变容器）     |
| 能否被 del 删除  | 合法的标识符本身删不掉，只能删除它绑定的对象       | 可以 del 变量，从而解除绑定                          |
| 典型例子         | `print`, `list`, `MyClass`, `os`, `MAX_VALUE`      | `a = 10` 中的 `a`，`name = "小明"` 中的 `name`       |

### 用代码最直观地演示区别

```python
# 以下这些都是合法的标识符，但还不是变量
def func(): pass        # func 是标识符（函数名）
class Person: pass      # Person 是标识符（类名）
import math             # math 是标识符（模块名）
MAX_RETRY = 5           # MAX_RETRY 是标识符，常量风格，但本质还是变量！

# 只有执行了赋值，才真正产生了「变量」
x = 100                 # ← 这一刻起，x 才真正成为一个变量
name = "Python"
lst = [1, 2, 3]

# 删除变量（解除绑定），标识符本身依然合法
del x
# print(x)  # NameError! x 这个标识符现在没有绑定任何对象，不能再当变量用了
```

### 再来一个经典的“陷阱题”

```python
a = 1000
b = 1000
print(a is b)      # 可能是 False！（大整数不缓存）

a = "hello_world"
b = "hello_world"
print(a is b)      # True（某些字符串会被实习）

# a 和 b 都是变量，但它们使用的标识符都是合法的标识符
```

### 一句话总结（面试必背）

> **标识符是给各种实体取的名字（语法规则），变量是运行时用标识符绑定到某个对象后形成的“名字-值”关系**。

记住下面这张图就再也不会混了：

```
标识符（名字） ───赋值──→ 变成变量
例子：   age                 age = 18
         ↑ 只是一串合法字符         ↑ 现在是变量，指向整数对象 18
```

所以：
- 所有变量的名字都是标识符  
- 但不是所有标识符都是变量（函数名、类名、模块名都不是变量）
